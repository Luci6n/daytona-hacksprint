# DaddyFix backend API integration guide

This document is the client contract for Kenji's `VisionService.swift` and
`VoiceManager.swift`. The canonical response models are defined in
`backend/models.py` and mirrored in
`DaddyFix/DaddyFix/Models/AnalysisResult.swift`.

## Base URL and interactive documentation

Local development base URL:

```text
http://127.0.0.1:8000
```

Replace it with the public Daytona/proxy URL during device integration. The
API currently has no client authentication. Sponsor credentials remain on the
server and must never be sent by the iOS app.

- Swagger UI: `GET /docs`
- OpenAPI JSON: `GET /openapi.json`
- API version: `0.2.0`

## End-to-end iOS flow

1. Swift captures a JPEG frame and optional device hint.
2. Apple Speech converts microphone audio into a transcript on the iPhone.
3. `VisionService.swift` sends the JPEG and transcript to `POST /analyze`.
4. The backend returns `AnalysisResult`; ARKit consumes `arAnnotations`.
5. Swift sends a selected repair instruction to `POST /speech/synthesize`.
6. The endpoint returns a WAV generated by Qwen3-TTS on Nosana; Swift plays it.

Apple Speech recognition is an iOS responsibility. Do not upload raw microphone
audio to `/analyze`; send the recognized text in `symptom`.

Use this before the demo and when selecting a backend URL.

```json
{
  "status": "ok",
  "service": "daddyfix-backend",
  "environment": "development",
  "demoMode": true,
  "providers": {
    "daytona": true,
    "kimi": false,
    "nosana": true,
    "nosanaTts": false,
    "oxylabs": true,
    "doubleword": true,
    "aiand": true
  }

```

Provider values indicate configuration presence, not a live network probe.
No keys, passwords, or endpoint tokens are returned.

## `POST /analyze`

Analyzes a captured repair scene and returns the shared `AnalysisResult`.

### Request

`Content-Type: application/json`

```json
{
  "symptom": "No hot water",
  "deviceHint": "Rinnai tankless water heater",
  "imageBase64": "data:image/jpeg;base64,/9j/4AAQSk..."
}
```

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `symptom` | string or null | No | Apple Speech transcript or typed symptom. |
| `deviceHint` | string or null | No | A visible brand/model hint improves manual search. |
| `imageBase64` | string or null | Live mode: yes | A data URL is recommended; raw base64 is also accepted as JPEG. |

In `DEMO_MODE=true`, all fields are optional and the deterministic water-heater
fixture is returned without spending provider credits.

For iOS, resize the captured image before encoding. A JPEG around 1280 px on its
long edge is sufficient for the demo and keeps request latency manageable.

### Live provider paths

- Moonshot path: Oxylabs context plus the image is sent directly to Kimi K2.6.
- ai& path: Doubleword Qwen3-VL first creates a visual observation; ai& Kimi
  K2.7 Code then reasons over that observation and the Oxylabs context.
- When Doubleword is configured, the final repair guidance receives an
  additional safety audit.

The backend never silently replaces a failed live call with demo data.

## `POST /speech/synthesize`

Generates the configured DaddyFix voice through Qwen3-TTS VoiceDesign on Nosana.

### Request

```json
{
  "text": "Easy, sweetheart. Turn off the breaker first."
}
```

### Successful response

- Status: `200`
- Content-Type: `audio/wav`
- Body: binary WAV data

Swift should store the response in `Data` and play it using `AVAudioPlayer`.
Do not attempt to decode this endpoint as JSON.

The voice description, language, and model are server configuration. The iOS
client deliberately cannot override the persona prompt.

## Error contract

All JSON errors use this shape:

```json
{
  "detail": "Human-readable explanation"
}
```

| Status | Meaning | Suggested iOS behavior |
| --- | --- | --- |
| `422` | Invalid request or missing live-mode image | Ask for another capture or correct the request. |
| `502` | A provider failed, returned invalid data, or rejected unsafe guidance | Show the error and allow retry; do not invent guidance. |
| `503` | Required credentials or Nosana TTS URL are not configured | Disable the affected action and show backend unavailable. |

Use a client timeout of at least 60 seconds for `/analyze` and 120 seconds for
the first `/speech/synthesize` call because cold GPU model loading can be slow.
Warm the TTS service before the live demo.

## Swift client contract sketch

Kenji owns the actual `VisionService.swift`; this sketch documents the wire
format without modifying that file.

```swift
struct AnalyzeRequest: Encodable {
    let symptom: String?
    let deviceHint: String?
    let imageBase64: String?
}

let payload = AnalyzeRequest(
    symptom: transcript,
    deviceHint: "Rinnai tankless water heater",
    imageBase64: "data:image/jpeg;base64," + jpegData.base64EncodedString()
)

var request = URLRequest(url: baseURL.appendingPathComponent("analyze"))
request.httpMethod = "POST"
request.setValue("application/json", forHTTPHeaderField: "Content-Type")
request.timeoutInterval = 120
request.httpBody = try JSONEncoder().encode(payload)

let (data, response) = try await URLSession.shared.data(for: request)
let analysis = try JSONDecoder().decode(AnalysisResult.self, from: data)
```

Check the HTTP status before decoding `AnalysisResult`; non-2xx responses use
the error shape above.

## Local verification

From the repository root:

```powershell
.\.venv\Scripts\python.exe -m uvicorn backend.main:app --reload --port 8000
```

In another PowerShell terminal:

```powershell
Invoke-RestMethod http://127.0.0.1:8000/health

$body = @{ symptom = 'No hot water' } | ConvertTo-Json
Invoke-RestMethod `
  -Method Post `
  -Uri http://127.0.0.1:8000/analyze `
  -ContentType 'application/json' `
  -Body $body
```
